# Phase 7 チーム

フェーズ7(Islands / 部分ハイドレーション + エコシステムプリミティブ + オーサリング
ツール)を進めるための 3 ロール構成。各エージェントは `.claude/agents/` のサブエージェント
として呼び出す。

| ロール | エージェント | 仕事 |
| --- | --- | --- |
| タスク分解する人 | [`phase7-planner`](./phase7-planner.md) | ロードマップ/設計メモを根拠に Phase 7 を着手可能な単位へ分解(依存・順序・DoD・検証手順)。コードは書かない。 |
| タスクを進める人 | [`phase7-implementer`](./phase7-implementer.md) | 切られたタスクを 1 つずつ実装し、テスト・型・カバレッジ・example ビルドまで緑にする。 |
| レビューする人 | [`skeptical-reviewer`](./skeptical-reviewer.md) | 完了報告の直前に、要件充足・規約準拠・副作用を「証拠」で検証し 🔴/🟡/🔵 で指摘。 |

## 流れ

```
phase7-planner → phase7-implementer →(完了報告の直前に必ず)skeptical-reviewer
                       ↑___________________ 🔴 を修正して再実装 ___________________|
```

1. **分解**: `phase7-planner` が Phase 7 のタスクカード(対象レイヤー / DoD / 検証手順 /
   依存)を作る。
2. **実装**: `phase7-implementer` が 1 タスクずつ仕上げる。core はランタイム非依存、
   ゼロ依存、`*.spec.ts` 100% カバレッジを死守。
3. **レビュー**: 完了報告の直前に `skeptical-reviewer` を必ず実行(`CLAUDE.md` の必須
   ワークフロー)。🔴 は全て修正、🟡 は対応か明確な理由付け。クリーンになるまで実装へ戻す。

全ロール共通の品質バー: ゼロ依存(`@types/bun` 以外禁止)/ `packages/core` ランタイム
非依存 / 全ソース 100% カバレッジ / `tsc` クリーン / ドキュメント二言語同期(EN+JA)。
